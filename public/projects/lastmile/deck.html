02 · 作品是什么

一个面向售前与 POC场景的 AI 评估平台

以客户的打分数据为唯一输入，自动产出"产品效果 / 使用策略 / 客户匹配度"三位一体的标准化评估报告。 让售前与解决方案同学，在客户现场直接完成原本需要数科与产品团队数周才能交付的工作。

INPUT

打分数据

客户提供样本评分文件 / 自有数据 / 内置案例数据三选一。

OUTPUT · 1

产品效果

命中率、提升度、KS、AUC 等核心指标的自动测算与可视化。

OUTPUT · 2

使用策略

阈值切分、分层规则、人群圈选建议，可直接落地业务系统。

OUTPUT · 3

客户匹配度

量化客户场景与产品能力的契合度，回答"适不适合用"。

04 · 解决什么问题

售前链路上两个长期未被解决的卡点

01

售前阶段：经典案例没有有效的展示方式

依赖静态 PPT 与口述讲解，客户难以直观感受产品在真实场景下的价值。"讲得再好，不如跑一次数。"

PPT × 口述
现状交付物
02

POC 阶段：评估周期长、口径不统一

客户给数据后，需要数科与产品团队人工跑数、对齐口径、出具报告； "效果 / 策略 / 匹配度"三个核心问题往往拖到数周后才能给出，商机持续流失。

2–4 周
单次评估耗时

本质矛盾： 售前节奏是"分钟级"的对话，但产品评估是"周级"的工程 —— 节奏错配，转化即流失。

05 · 核心设计思路

把"评估流水线"产品化，让一个人就能跑完

传统做法是"数据 + 数科 / 产品同学 + 脚本笔记本"。我们把这条流水线沉淀成可配置的引擎，让售前一人即可完成端到端交付。

交互层
统一 Web 控制台 (Streamlit) · 案例库浏览 · 数据上传 · 配置面板 · 报告导出。 面向售前与客户，零代码使用。
AI 层
策略机器人 + 叙事生成 · 自动给出阈值与分层建议 · 用业务语言解释指标含义， 替代"看图说话"的人工解读。
引擎层
Report Pipeline 评估引擎 · 规则翻译 → 指标计算 → 图表构建 → 叙事生成 → 导出。 一份配置驱动全流程，可复用、可审计。
数据层
案例库 + 数据隔离机制 · 内置经典客户案例 · 客户数据本地化运行， 满足金融级合规要求。
06 · 功能操作流程

从打开浏览器到拿到报告，五步走完

01

选择场景

售前演示 → 调用案例库；POC 评估 → 上传客户数据。

02

配置参数

标签字段、目标变量、阈值范围、业务规则等，可视化勾选。

03

一键运行

引擎自动完成指标计算、图表构建、策略生成。

04

AI 叙事

用业务语言自动生成结论与建议，可直接对客解读。

05

导出交付

HTML 报告 / PDF / 策略配置文件，多格式一键下载。

SAFETY

数据隔离

客户数据本地运行，全流程不出域，满足合规要求。

CONSISTENCY

口径统一

所有评估走同一套引擎，杜绝"人不同结论不同"。

TRACEABILITY

可追溯

每次运行保存完整配置与结果，方便后续复盘与审计。
